## 任务物品掉落

`[seeking]` 的 `[int data]` 写任务物品 ID/数量；如果同文件存在 `[monster reward item]`，它提供接任务后的任务物品掉落候选。

`[monster reward item]` 的静态行形是 7 列一组：

| 列 | 静态含义 |
| ---: | --- |
| 1 | 怪物/目标 ID |
| 2 | 副本 ID |
| 3 | 难度 |
| 4 | 物品 ID |
| 5 | 数量 |
| 6 | 概率候选 |
| 7 | 限制/控制候选 |

主目标中任务怪物奖励物的物品 ID 全部能闭合到物品 registry；副本 ID 全部能闭合到 `dungeon/dungeon.lst`。少量怪物/目标 ID 不在 `monster.lst`，遇到时不要强行当普通怪物解释。

## 禁止外推

- 不把 `[type]` 名称写成完整运行语义。
- 不把 `[sub type]` 数字跨类型复用。
- 不把 `[int data]` 裸数字按外形猜 registry。
- 不把静态任务条件写成实机任务完成、计数成功、掉落成功、奖励发放成功或 PVP 最终规则。
