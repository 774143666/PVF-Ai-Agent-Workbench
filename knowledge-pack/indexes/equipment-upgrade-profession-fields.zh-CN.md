# Equipment 升级与副职业字段索引

## 用途

本索引用于只读判断 equipment `.equ` 中强化或增幅条件、账号绑定线性成长、强化概率与费用、专业职业限制、专业产出修正、经验加成和职业技能门槛字段。它记录目标 PVF 的结构事实，不授权直接写 PVF。

## 总规则

- 以下字段默认状态均为 `需验证`。
- 强化概率、费用折扣、经验和专业产出数值存在不同刻度，不能把原始数值直接显示成百分比。
- 闭合块必须连同结束标签读取；数字 ID 必须按字段上下文选择 registry。
- 教程中的职业等级或百分比只能作为线索，目标 PVF 的实际列形和数值优先。

## 强化、增幅与账号成长

| 字段 | 目标列形 | 读取边界 |
| --- | --- | --- |
| `[equipment upgrade]` | 闭合的一组或两组四参数记录 | 位于 `[if]`。每组按“强化类型 token、比较符、等级阈值、尾随整数”读取，常见类型为 `upgrade`、`amplify`。必须读取 `[/equipment upgrade]`；尾随整数及多组之间的组合逻辑未定性。 |
| `[account bind linear ability]` | 闭合的嵌套成长表 | 根级容器，目标以 `[grade]` 等级范围开头，后接若干属性及起止数值。属性字段和列数随装备类型变化，必须读取 `[/account bind linear ability]`，不能套用固定子字段模板。 |
| `[upgrade prob increase]` | 单数值 | 强化成功率增加线索。目标描述与原始值显示存在倍率刻度，例如描述 10% 的样本值为 `10000`；未经目标公式验证不得直接按百分比显示。 |
| `[upgrade cost discount]` | 单小数 | 强化费用折扣线索；目标值为 `5.0` 或 `30.0`，描述文本与百分比相符，但结算范围和叠加方式仍需验证。 |
| `[assault cost discount]` | 单小数 | 街头争霸或 assault 费用折扣线索；目标唯一值为 `5.0`。不要与强化费用字段混用。 |
| `[item overpower part]` | 无值存在标记 | 套装或部件强化相关标记；目标无闭合标签，精确作用和与 `[set ability]` 的关系未由静态文件闭合。 |

`[equipment upgrade]` 中目标比较符可见 `=`、`>=`、`>`、`<`、`=<`。必须保留目标原始拼写，不自行改写比较符。

## 副职业限制与产出修正

| 字段 | 目标列形 | 读取边界 |
| --- | --- | --- |
| `[expertjob only]` | 闭合的“副职业 token、等级” | 目标 token 为 `alchemist`、`disjointer`、`doll_controller`、`enchanter`，目标等级均为 `1`。必须读取 `[/expertjob only]`。 |
| `[prof compound rate]` | 单数值 | 专业制作成功率或合成率加成线索；目标值为 `5`，精确刻度和适用专业由相邻 `[expertjob only]` 决定。 |
| `[prof result variation]` | 两个数值 | 专业制作结果变化线索；目标固定形态为 `5 200`，可作为概率与结果倍率候选，但两列精确公式需实机确认。 |
| `[prof disjoint result variation]` | 单数值 | 分解结果变化线索；目标唯一值为 `10`，实际数量换算需验证。 |
| `[prof material variation]` | 单数值 | 专业材料消耗变化线索；目标唯一值为 `-10`，负号必须保留。 |
| `[prof additional gain exp]` | 两个数值 | 副职业额外经验获得线索；目标唯一形态为 `10 1`，两列分别的概率、经验或次数语义未完全闭合。 |

副职业修正字段必须与同文件的 `[expertjob only]` 一起读取，不能脱离专业 token 单独套用。

## 经验、活动与技能门槛

| 字段 | 目标列形 | 读取边界 |
| --- | --- | --- |
| `[exp advantage]` | 单数值 | 地下城击杀经验加成线索；目标值为 `10`，目标说明文本对应 10%，但最终结算和叠加需实机确认。 |
| `[exp]` | 单数值 | 经验相关单值字段；目标唯一值为 `50`，同文件说明文本写 10%，因此不能按原始值直接显示百分比。 |
| `[user power war point]` | 单数值 | 势力战或 power war 点数线索；目标值有 `3` 与 `3000` 两种刻度，具体单位和活动版本未定性。 |
| `[required job skill]` | 闭合的“职业 token、技能 ID”对列表 | 技能 ID 必须按职业选择 skill registry。目标唯一记录中，鬼剑士与黑暗武士的 ID `33` 均解析为“光剑掌握”。必须读取 `[/required job skill]`。 |

## 最低验证清单

1. `[equipment upgrade]`、账号成长、副职业限制和技能门槛必须检查闭合。
2. 强化类型、比较符和尾随整数按原形保留。
3. 概率、经验、费用和专业产出字段不得统一套用同一百分比刻度。
4. 副职业修正与 `[expertjob only]` 同文件读取。
5. `[required job skill]` 的每个 ID 按紧邻职业 token 选 skill registry。
6. 实际强化、费用、经验和专业产出仍需游戏内验证。
