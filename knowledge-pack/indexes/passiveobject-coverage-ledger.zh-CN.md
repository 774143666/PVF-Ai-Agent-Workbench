# PassiveObject / AttackInfo / Hitbox 覆盖账本

状态：需验证

本文记录主目标 PVF 中 `passiveobject/` 主线的只读覆盖进度。它用于告诉 Agent 下一步查哪里、哪些块已经有主目标样本、哪些结论仍不能由静态只读证明。

治理提示：本账本已超过约 100KB，后续新增小桶只写短结论、边界和路由行；长坐标、长创建链、同族批量差异写入分片索引。落账前先读 `passiveobject-ledger-governance.zh-CN.md`。

## 入口规则

| 问题 | 结论 |
| --- | --- |
| 数字 passiveobject ID 怎么查？ | 先走 `passiveobject/passiveobject.lst`。不要用数字形状猜类型。 |
| 辅助对照能不能写成正式结论？ | 不能。辅助对照只能提示差异和补充观察，主目标结论必须回主目标单独复核。 |
| `.obj -> .act/.ani/.atk` 怎么跟？ | 从当前 `.obj` 所在目录解析相对路径；`.act` 中的 ANI 引用继续按 `.act` 目录解析。 |
| Hitbox 能不能只靠 `.atk` 判断？ | 不能。`.atk` 描述攻击信息；命中/攻击框需要进入对应 `.ani` 的帧数据查看。 |
| `SearchScript` 未命中能否证明不存在？ | 不能直接证明。二进制 ANI、路径大小写、索引行为都可能影响搜索结果。 |
