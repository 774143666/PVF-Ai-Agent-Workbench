# 技能派生与强制取消流程

状态：默认可用

## 适用

让 A 技能在特定状态触发 B 技能，或允许平砍、技能中强制使用另一技能。

## 只读闭合

1. 确认目标 PVF。
2. 定位角色职业和 skill registry。
3. 解析 A 技能 ID 和 `.skl`。
4. 解析 B 技能 ID 和 `.skl`。
5. 如需解释参数列，先查 knowledge-pack 内置结构化参数事实表取得 `[static data]` / `[level info]` / `[level property]` 的候选列义。
6. 读取目标 `.skl`，核对结构化事实表候选列义对应的实际列数、列顺序和上下文。
7. 查 A 技能 state、substate、动作、NUT 入口和 static data。
8. 查 B 技能 state、substate 和释放条件。
9. 如果涉及 NUT，查询目标脚本入口和 API，不猜函数名。
10. 标出哪些参数是目标 PVF 已确认，哪些只是结构化事实表或辅助索引线索，哪些仍需验证。

## 写入流程

1. 用户明确授权写 PVF。
2. 明确源 PVF 和输出 PVF。
3. 备份源 PVF。
4. 只改目标技能链必需文件。
5. 保存输出 PVF。
6. 重新打开输出 PVF 读回改动文件。
7. 输出变更清单。

## 游戏内验证

- A 技能能正常释放。
- 指定状态下能触发或强制使用 B 技能。
- 非指定状态不乱触发。
- 技能不会卡死、黑屏、崩溃或长期锁状态。
- 联动有伤害或表现要求时，确认命中、伤害、动作和冷却。
