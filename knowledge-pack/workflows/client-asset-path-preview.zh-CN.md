# 客户端资源路径预览流程

状态：默认可用

## 适用

用于只读检查 PVF 中引用的 `.ani`、`.als`、`.img`、`.act`、`.til`、`.ui`、SoundPacks、Music 等客户端资源路径，并提示 ImagePacks2 / NPK 风险。

本流程不写 NPK，不加入、删除、去重或合并 IMG，不证明客户端资源实际完整。

## 输入

- PVF 文件、提取目录或 planner 输出。
- 可选客户端 `ImagePacks2`、SoundPacks、Music 目录。
- 可选资源路径关键词或文件类型过滤。

## 执行

1. 先确认任务是资源路径预览，不是资源写入或打包。
2. 从 PVF 文本或提取 manifest 中收集 `.ani/.als/.img/.act/.til/.ui` 等引用。
3. 对 UI、装备、光环、时装、宠物、副本、monster、passiveobject 分别标记来源类型。
4. 如果提供客户端资源目录，只做存在性或候选容器检查。
5. 对缺失路径、大小写差异、路径别名、外部容器、未支持格式全部写入风险项。
6. 输出资源路径 preview index。

## 输出审阅

- 资源引用路径列表。
- 来源 PVF 文件类型。
- 是否提供客户端目录。
- 命中/未命中/疑似外部容器。
- 光环、时装、宠物、副本 UI、地图贴图、音频的专项风险。

## 验收

- 输出明确标记未写 NPK、未改客户端。
- 未命中资源没有被静默忽略。
- PVF 引用和客户端存在性分开报告。
- 涉及显示或音频时标记需要实机验证。

## 边界

- Script.pvf 引用不证明客户端资源存在。
- NPK/ImagePacks2 命中不证明图层、动作、帧序列或 UI 运行时正确。
- 光环/时装外观需要穿戴实机验证。
- 副本地图贴图、机关、UI、音频需要进图实机验证。
- 写 NPK、合并 IMG、删除 IMG 必须另行授权并走生产生命周期。
