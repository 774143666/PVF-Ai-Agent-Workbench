# PVF Registry / LST Topology 术语字典

状态：默认可用

用途：解释 `.lst` registry、ID、路径闭合、未注册文件和孤儿风险的口径。本文不授权写 PVF。

## 核心术语

| 术语 | 静态含义 | 主目标观察 | 边界 |
| --- | --- | --- | --- |
| `.lst` | ID 到值的列表文件 | 主目标 83 个可读 `.lst` | 不是所有 `.lst` 都是文件路径 registry |
| 文件路径 registry | ID 到 PVF 内部文件路径的列表 | 主目标 62 个 | 可用于解析 ID 到路径，但不证明实机加载成功 |
| 名称/数值表 | ID 到显示名、文本、数值组或任务组的列表 | 主目标 19 个 | 不能拿来判断文件路径存在性 |
| 空 `.lst` | 文件可读但没有可解析注册项 | 主目标 2 个 | 不等于错误，需看所属系统是否引用 |
| 注册项 | 一行 ID + 反引号内容 | 文件路径 registry 共 102778 条 | 反引号内容要先判断是不是路径 |
| 注册路径存在 | registry 解析出的 PVF 内部路径能在文件表中命中 | 102361 条 | 只证明静态文件存在 |
| 注册路径缺失 | 文件路径 registry 指向的 PVF 内部路径未命中 | 417 条 | 多见于历史/跨语言/旧 registry；不自动证明当前玩法断链 |
| 重复 ID | 同一 registry 内多个条目使用同一 ID | 文件路径 registry 未观察到 | 跨 registry 重复 ID 很常见，不算同表重复 |
| 重复路径 | 同一 registry 内多个 ID 指向同一路径 | 8 个 registry 观察到 | 可能是复用资源或多入口映射，不自动是错误 |
| 跨 registry ID 重复 | 同一数字出现在多个 registry | 文件路径 registry 6255 个数字重复 | 这是“不能裸猜数字”的核心证据 |
| 未注册文件 | 未被文件路径 registry 直接指向的文件 | 302882 个 | 不等于孤儿，可能由脚本、动作、资源链或客户端链引用 |

## 解析规则

| 场景 | 正确动作 | 禁止动作 |
| --- | --- | --- |
| 字段里出现数字 | 先确认父块和字段，再选择 registry | 直接全局搜索数字后下结论 |
| 商店商品 ID | 按商品上下文解析 equipment / stackable / creature 等 registry | 只看数字大小猜类型 |
| 怪物 ID | 按 monster registry 解析 | 拿同数字去套 map、quest 或 passiveobject |
| 地图/副本 ID | 按 map / dungeon registry 解析 | 把地图 ID 当副本 ID 或任务 ID |
| 技能 ID | 按职业 skill registry 解析 | 把不同职业同数字技能混用 |
| 名称表 ID | 按文本/名称用途解释 | 当作 PVF 文件路径检查 |

## 静态与动态边界

静态只读可以确认：

- 哪些 `.lst` 可读。
- 哪些 `.lst` 是文件路径 registry。
- 某个 registry 内 ID 是否唯一。
- 某个 registry 指向的路径是否存在。
- 哪些路径重复、哪些数字跨 registry 重复。

静态只读不能确认：

- 实机是否加载该文件。
- UI、音频、动画、任务、怪物、技能或商店是否正常。
- 客户端 ImagePacks2 / NPK 资源是否完整。
- 服务端是否放行。
- 未注册文件是否可删除。
