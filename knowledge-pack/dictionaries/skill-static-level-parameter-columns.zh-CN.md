# 技能静态 / 动态参数列词典

状态：需验证

## 适用

用于解释技能 `.skl` 中参数列的查询方式，尤其是 `[static data]`、`[level info]`、`[level property]` 的关系。

本文只说明如何读列，不定义某个技能的最终运行效果。

## 字段

### `[static data]`

状态：需验证

含义：技能的静态参数表。知识包结构化参数事实中的“静态 n”通常指向该表的候选列号。

注意：目标 `.skl` 中必须实际存在对应列。列号、列数和上下文不一致时，结构化参数事实表只能作为线索。

### `[level info]`

状态：需验证

含义：技能随等级变化的参数表。知识包结构化参数事实中的“动态 n”通常指向该表的候选列号。

注意：同一技能在不同 PVF 中可能有不同等级列或补充列。写入前必须读取目标 PVF。

### `[level property]`

状态：需验证

含义：技能等级属性说明或列映射说明。文本中出现的静态 / 动态引用可用于定位候选列。

注意：说明文字不证明真实效果；必须回到目标 `.skl` 和运行入口验证。

## 最低核查

1. 先按职业 skill registry 解析目标技能路径。
2. 再查 knowledge-pack 内置结构化参数事实的技能名、职业目录和路径是否匹配。
3. 读取目标 `.skl` 的 `[static data]`、`[level info]`、`[level property]`。
4. 核对目标列是否存在，不能只按 结构化事实表修改。
5. 涉及 state、substate、NUT、passiveobject、冷却、命中、伤害或手感时，另走技能 runtime 核查。
6. 存在 TP/特性/Ex 技能时，同时读取对应 `*ex.skl` 或技能树指向的特性技能；基础 `.skl` 修改不保证点 TP 后仍生效。
7. 涉及连射、按住键、蓄力、爆炸次数、视觉范围或动作速度时，至少拆分“不点 TP”和“点 TP”两组实机验证。

## 禁止外推

- 不把某个技能的列义推广到同职业全部技能。
- 不把结构化参数事实表的列说明当成目标 PVF 已验证结论。
- 不从列名或说明文字直接推断伤害、范围、持续时间或异常状态行为。
- 不把说明文字中的“范围”“大小”“速度”等词直接当成视觉效果；视觉不变时继续检查 passiveobject、ANI/ATK 或客户端资源。
- 不跳过目标 PVF 读回和必要实机验证。
